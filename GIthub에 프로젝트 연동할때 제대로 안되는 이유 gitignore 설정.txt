처음 github에서 리포지토리 생성하고 빈 폴더에 git clone (주소) 하면 
아무것도 없는 파일에 .git폴더와 .gitignore 파일 딸랑 2개 있을것이다.
이 gitignore 파일이 없으면 commit이 안된다. 자세한 이유는 나중에 공부해야한다. 

< 문제점 >
중요한건, 이 gitignore 파일에서 기술하고있는 파일제외목록이 내 프로젝트 프로그램을 실행하는데
필수적인 여러 파일들을 트래킹하지 않게해서 
프로젝트를 commit 하고 push한다음 pull하면, 
파일 몇개가 사라져있어서 프로그램이 제대로 실행되지 않는다.

< 어떻게 해결해야하는가 >
gitignore 파일에서, 내 프로그램 실행에 필수적인 파일들은 ignore리스트에서 삭제하고,
결과적으로 git이 파일들을 트래킹하게 하면 된다.

< AR49 Direct2D 에서 제외됬던 파일이나 폴더 목록들>
.pdb
.user 
bin 

파악한건 요 3개이고, 그중 .user 와 bin을 gitignore 파일에서 제거해주니 
commit, push, pull 해서 새로 받은 프로젝트가 정상적으로 잘 실행됨.

 * 특히 bin 폴더 안에는 여러 dll파일들이 있었고, 분명 push할땐 dll파일이 잘있었는데
   pull할땐 bin폴더 통째로 사라져서 프로그램이 제대로 실행안되는 골치아픈 일이 있었다.

 * .user가 뭐하는 놈인지 모르겠다. 처음엔 .user 와 bin 둘다 삭제 안하고 dll이 없어서 프로그램이 안되나 싶어서
   bin만 gitignore에서 삭제했는데
   그러면 빌드하면 쉐이더파일이 없다, 에셋없다라고 하고 프로그램이 실행안됬음.
   .user 까지 제외목록에서 없애주니까 프로젝트 pull한게 잘 실행됬음

 *참고 .pdb 파일을 ignore리스트에서 없애버리면 commit은 되는데 push가 안됨.. 
  오류나서 출력 확인하라고 해도 아무것도 안뜸. 왜 이러는지는 공부해야함 이유를 모르겠음
  어쨌든 프로그램 동작 잘되는데 불필요하게 .pdb를 ignore리스트에서 굳이 없앨 필요는 없다. 다행이다.

  (bin은 .[Bb]in 요딴식으로 되어있던걸로기억.. gitignore 규칙도 공부했었음 이거 안했으면
   bin 어딨지 하면서 한참 헤멨을 거임)


!! 결론 - github써서 AR49 Direct2D 프로젝트 문제없이 동작하게 하려면
           gitignore 파일에서 .user 와 bin 이 적힌 부분을 없애준다.
